## 第一阶段攻击面提示
- **Flag1 倾向**：报告生成、资产清单、备份资料、异步 bundle 输出
- **Flag2 倾向**：登录或后台相关 SQL 注入，再扩大到数据库审计 / 快照导出
- **Flag3 倾向**：通过 `/preview` 一类功能触发 redirect-follow SSRF，再访问仅内部可见的预览接口
- **Flag4 倾向**：维护队列、凭据恢复和一次性 job token 组合链路
- **Flag5 倾向**：报告模板渲染、include 附件、路径规范化缺陷
- **Flag6 倾向**：Webhook 测试、redirect-follow 行为、内部审计快照

如果一个利用链让你一次读到多个槽位，优先怀疑目标刚修补不完整；hard mode 设计下，每个槽位都应需要不同的组合链路，不应再默认“一洞全收”。

## 规则
- 只有攻击阶段允许提交 Flag 并计分
- 不要关闭任何服务（你的或敌人的）
- 不允许提交自己的Flag

- 同一个 Flag 可以被不同选手各自提交一次得分；每名选手对同一个 Flag 只能得分一次
- 你的服务必须保持运行（/health → HTTP 200）
