# 安全加固记录（本轮）

> 配套文档：`SECURITY_AUDIT.md`（完整审查报告）。本文件只记录本轮实际落地的改动与待办。

## 一、已落地改动

### M1 — 容器 PID 限制（防 fork 炸弹）
所有 `containers.run(...)` 补 `pids_limit=512`，防选手 Agent 在容器内耗尽宿主 PID 造成跨队 DoS。

覆盖 5 处：
- `referee-engine/main.py` — werewolf-agent、target、agent
- `orchestrator/round_orchestrator.py` — target、agent

### M2 — Agent 容器能力收紧
仅对**对抗性 agent 容器**加 `cap_drop=["ALL"]` + `security_opt=["no-new-privileges:true"]`：
- `referee-engine/main.py` — werewolf-agent、agent
- `orchestrator/round_orchestrator.py` — agent

**target 容器有意豁免**：其 entrypoint 跑 `sshd` / `chpasswd` / setuid 登录（`target-image/docker-entrypoint.sh`），SSH 是该 Juice Shop 靶机的核心攻击面。无脑 drop 会打断 sshd 启动和选手登录，并可能阉割靶机里故意设计的提权挑战。target 仅加了 `pids_limit`。

### M4 — 裁判 API 端口绑回环
`docker-compose.yml`：`"8000:8000"` → `"${REFEREE_BIND_HOST:-127.0.0.1}:8000:8000"`

前端走 docker 内网 `referee-engine:8000` 访问后端（`frontend/nginx.conf`），不依赖宿主发布端口，故不受影响。需要对外暴露时设 `REFEREE_BIND_HOST=0.0.0.0`。

### M3 — 前端管理密钥改 sessionStorage（方案 A）
`localStorage` → `sessionStorage`，管理总钥不再跨标签页/重启持久化，关页即清。
- `frontend/src/api.ts:27`
- `frontend/src/components/Layout.tsx`（读取 + 写入两处）

> 说明：当前前端**无 XSS sink**（无 `dangerouslySetInnerHTML`/`innerHTML`/`eval` 等），M3 属纵深防御/防回归，非当前可利用漏洞。若将来引入富文本/markdown 渲染，再考虑升级为会话令牌化（方案 B：仿 `/api/ws-ticket` 短时 token）。

### 附带修复 — ArenaPage.tsx 既有 TS 错误
全项目 `tsc` 照出的既有错误（非本轮改动引入），已修：
- `WerewolfState` 补 `board_label?: string | null`
- 3 处 `toNumber(...)` 加 `?? null`，将 `number | undefined` 收口为 `number | null`

`tsc --noEmit` 现已全绿。
